## Phase 1：基础设施

### Step 1.1 — PasswordManager.kt（Kotlin 新建）

**文件**：`src/com/android/calculator2/PasswordManager.kt`

**职责**：
- 单例模式，SharedPreferences 存取密码
- `checkPassword(input): Boolean`
- `getPassword(): String` / `setPassword(pwd: String)`
- 默认密码 `"1234"`
- 首次启动时初始化默认密码

**依赖**：无外部依赖，纯 SharedPreferences

---

### Step 1.2 — VaultRepository.kt（Kotlin 新建）

**文件**：`src/com/android/calculator2/vault/VaultRepository.kt`

**职责**：
- 私有目录管理：`/data/data/com.android.calculator2/files/vault/`
- `initVault()` — 创建目录结构（images/、videos/）
- `importFromGallery(uris)` — 从 ContentResolver 复制文件到私有目录
- `exportToGallery(items)` — 复制到 MediaStore 公共目录
- `deleteItems(items)` — 删除私有目录中的文件
- `getAllMedia(): List<MediaItem>` — 扫描私有目录返回媒体列表
- `getThumbnail(path, type): Bitmap` — 用 ThumbnailUtils 生成缩略图

**依赖**：Android SDK 原生 API（ContentResolver、MediaStore、ThumbnailUtils）

### Step 1.3 — MediaItem.kt（Kotlin 新建）

**文件**：`src/com/android/calculator2/vault/MediaItem.kt`

**职责**：数据模型

## Phase 2：核心入口改造

### Step 2.1 — CalculatorExpr 新增判断方法（Java 修改）

**修改**：`src/com/android/calculator2/CalculatorExpr.java`

新增方法：
```java
public boolean isPureNumeric() {
    // 判断表达式中是否只包含数字常量 token
    // 无运算符、无函数、无括号 → 返回 true
}

public String toDisplayString() {
    // 将当前表达式转为纯数字字符串，如 "1234"
}
```

这是密码检测的前提——需要知道用户输入的是不是纯数字。

---

### Step 2.2 — Calculator.onEquals() 密码注入（Java 修改）

**修改**：`src/com/android/calculator2/Calculator.java` 的 `onEquals()` 方法

在现有求值逻辑之前插入密码检测分支：

```
onEquals() 调用时：
  1. 检查 isPureNumeric()
  2. 如果是纯数字 → 取出数字字符串 → 调用 PasswordManager.checkPassword()
  3. 密码匹配 → Intent 跳转 VaultActivity → 清空输入框 → return
  4. 密码不匹配或非纯数字 → 走原有正常计算逻辑（无感知降级）
```

**关键点**：
- 密码错误时**不做任何特殊提示**，继续走原逻辑显示结果
- 纯数字如 `5678=` 会得到结果 `5678`，看起来像普通计算器行为
- 外人完全无法察觉异常
